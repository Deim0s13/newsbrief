### üìö 5. Feature Branch Workflow for v0.6.0 - **DOCUMENTED**

**Status**: ‚úÖ Already documented in `docs/BRANCHING_STRATEGY.md`

**Workflow Summary**:
1. **Create feature branches from `dev`**: `git checkout -b feature/issue-XX-description`
2. **Work and commit**: Use conventional commit messages (`feat:`, `fix:`, etc.)
3. **Create PR to `dev`**: Review and merge feature ‚Üí dev
4. **Release PR to `main`**: When milestone complete, PR dev ‚Üí main
5. **Tag release**: `git tag -a vX.Y.Z -m "Release vX.Y.Z"`

**For v0.6.0**: Follow this workflow strictly
- All features developed in `feature/*` branches
- Merge to `dev` for integration testing
- Only merge `dev` ‚Üí `main` for final releases
